1. Как создать миграцию начального состояния для таблицы с партицированием.
Текущая схема работает корректно если использовать db.create_all()
Однако если использовать alembic, то для чистой таблицы
flask db init
flask db migrate
flask db upgrade
Создаются миграции, однако возникает проблема со вставкой данных в партицированную таблицу

Если изначально создать не партиицрованную таблицу (закомментировав ряд строк),
выполнить:
flask db init
flask db migrate
flask db upgrade
а потом добавить конфигурацию на партицирование, и выполнить
flask db migrate
flask db upgrade

Вставка работает, но таблицы не партицированные судя по pgadmin

В таком случае возникает в голове два варианта: либо как-то править файл миграции (как?),
или использовать для инициализации db.create_all(), а потом уже накатывать миграции через alembic
